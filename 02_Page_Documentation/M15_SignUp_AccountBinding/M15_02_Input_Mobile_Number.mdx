4. ## Input Box
    

```Plain
SIGNUP_BINDING_INPUT_Phone Input Field
```

### Key Function

Numeric input box for users to manually enter their mobile number with validation and error presentation.

  

See global components documentation for input box for more details. [LINK]

---

### Visual Reference

---

### Field List

|   |   |
|---|---|
|Field|Description|
|phone_value|Raw number string|
|is_valid|Derived validity result|

---

### Display Logic

- Default: neutral border and placeholder text in the input box
    
- Active: placeholder text becomes input box header in small text on top left.
    
- Invalid: red border + error text (see component 8).
    

### Interaction Logic

- When user taps to input their phone number, default phone numeric keypad is used to input the digits.
    
- Phone number validation is required.
    
- Validation on change/blur (format only, per Figma).
    

---

### CMS Field Specification

**(none)** — user-entered only.

7. ## Region Picker Modal
    

```Plain
GLOBAL_SELECT_Region Picker Modal
```

### Key Function

Full-screen searchable modal to select a country/region dialing code.

---

### Visual Reference

---

### Field List

|   |   |
|---|---|
|Field|Description|
|search_placeholder|Placeholder for search bar|
|popular_header_text|“Popular” label|
|region_items|Array of regions with `{name, code, right_index_letter}`|
|empty_state_text|Text for “No Results” state|

---

### Display Logic

- Search bar at the top.
    
- Alphabet index on the right.
    
- Popular list block at top, with China, HK, Macau set as the default phone number regions.
    
- Empty state with icon/message when there are no results.
    

### Interaction Logic

- Type to filter results; index disabled during search input.
    
- Tap a region → modal closes and updates `selected_region_code`.
