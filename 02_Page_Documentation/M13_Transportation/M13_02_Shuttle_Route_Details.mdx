## 1.  Schedule Selection Tab

```Plain
TRANS_FLOW_ScheduleSelectionTab
```

### Key Function

This component displays **different schedule groups or tabs** (e.g., All-day service, Peak hours, Specific dates), allowing users to **switch quickly** between various schedule options based on the search result.

### Visual Reference

To be added (Figma link)

### Field List

| Fields         | Description                                                                       |
| -------------- | --------------------------------------------------------------------------------- |
| schedule_title | The label text shown on each tab button, used to distinguish different schedules. |

### Display Logic

- The component is usually located **below the RouteSummaryCard**, serving as navigation before the actual timetable display.
- Based on the Shuttle Bus **departure and destination** selected in the `M13_01 Shuttle Bus` page, the corresponding route and available schedules are retrieved.

### Interaction Logic

- When the user clicks a different tab, the **TripDetailsStepper** or the following schedule list updates dynamically to show the schedule and description (`schedule_description`) associated with the selected `schedule_tab_title`.

### CMS Field Specification

#### `schedule_title`

Contains a JSON array of all configured schedule options, each object including the label title, description, and schedule list data.

| Specification Item | Details                                   |
| ------------------ | ----------------------------------------- |
| Field Type         | Text input (single line)                  |
| Max Length         | 20 characters                             |
| Characters Limit   | None                                      |
| Example            | Mon - Fri                                 |
| Input Validation   | Must not contain HTML or script tags.     |
| Localization       | Must support all display languages.       |
| Data Source        | Maintained in CCMS Transport Routes Table |
| Dependencies       | Used for LabelSelectionTabs tab text.     |

## 2.   Route Summary Card

```Plain
TRANS_DISPLAY_RouteSummaryCard
```

### Key Function

This card component presents a **concise summary** of the current route, including the **departure and destination stations** and the **estimated total journey time**.

### Visual Reference

To be added (Figma link)

### Field List

| Fields               | Description                                                                                                    |
| -------------------- | -------------------------------------------------------------------------------------------------------------- |
| station_name         | The names of the departure and destination stations.                                                           |
| schedule_description | Describes schedule details for the selected route, such as the first and last bus times and average frequency. |

### Display Logic

- Displays the starting and ending stations of the current route.
- Station names are displayed by referencing the `station_id`.

### Interaction Logic

- Clicking the switch button swaps the Departure and Destination, refreshing the corresponding `schedule_description`.

## 3.  Trip Details Stepper

```Plain
TRANS_DISPLAY_TripDetailsStepper
```

### Key Function

This component displays the **full list of route stops** in a **timeline or stepper format**, helping users clearly understand the travel route and sequence of stations.

### Visual Reference

To be added (Figma link)

### Field List

| Fields                | Description                                           |
| --------------------- | ----------------------------------------------------- |
| estimate_journey_time | Estimated one-way travel time (e.g., 30 minutes).     |
| station_name          | Unique identifier of each stop along the route.       |
| order                 | Sequential order number of the stop within the route. |

### Display Logic

- Stops are displayed vertically based on their `order` value to form the route timeline.
- Station names are displayed by referencing the `station_id`.
- Intermediate stops between the departure and destination are collapsed by default.

### Interaction Logic

- Clicking a stop name or icon may trigger the **HowtoFindModal**, which provides detailed directions for locating that stop OR launch WeChat default map

---

### CMS Field Specification

#### `estimate_journey_time`

| Localization       | Must support all display languages.            |
| Data Source        | Maintained in CCMS Transport Routes Table      |
| Dependencies       | Key information displayed in RouteSummaryCard. |
