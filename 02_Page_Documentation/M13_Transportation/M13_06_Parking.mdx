### Field List

| Fields            | Description                                                         |

## 5.  **CarparkList**

```Plain
TRANS_LIST_CarparkList
```

### **Key Function**

Displays a **list** of all eligible carparks and their **detailed information** to help users browse and select.

### **Visual Reference**

<FigmaEmbed url="https://www.figma.com/design/EHB41JQIomI0iwPWzaXAkw/-Dev--NGMP-UXUI-Designs?node-id=7729-15895&t=EIrrWlE3DFcM0Te3-4" />

### **Field List**

| Fields                   | Description                                                  |
| ------------------------ | ------------------------------------------------------------ |
| carpark_name             | The main name of the carpark.                                |
| carpark_card_description | A short description displayed in the carpark list card.      |
| parking_label            | The carpark’s custom label (e.g., indoor, outdoor, EV only). |

### **Display Logic**

- The list appears below the **All Available Carparks** section title at the bottom of the page.
- Each item (Carpark Card) corresponds to a carpark record and displays `carpark_name`, `carpark_card_description`, and `parking_label`.
- List order: Sorted by recommended order in the CMS.

### **Interaction Logic**

- Clicking a carpark card should navigate to that carpark’s detail page. See: [M13_07 Carpark](/02_Page_Documentation/M13_Transportation/M13_07_Carpark)

---

### **CMS Field Specification**

#### `carpark_name`

Displays the official or commonly used name of the carpark.

| Specification Item | Details                                                                             |
| ------------------ | ----------------------------------------------------------------------------------- |
| Field Type         | Text input (single line)                                                            |
| Max Length         | 40 characters                                                                       |
| Characters Limit   | None                                                                                |
| Example            | P1                                                                                  |
| Input Validation   | Must not contain HTML or script tags. Standard punctuation and numbers are allowed. |
| Localization       | Must support all display languages.                                                 |
| Data Source        | Maintained in CCMS.                                                                 |
| Dependencies       | Used as the primary identifier in carpark list views.                               |

---
#### `carpark_description`

Displays the short descriptive text of the carpark.

| Specification Item | Details                                                                    |
| ------------------ | -------------------------------------------------------------------------- |
| Field Type         | Text input (single line)                                                   |
| Max Length         | 200 characters                                                             |
| Example            | Direct to Opal Lobby, Jade Lobby, Pearl Lobby, Crystal Lobby, East Square. |
| Input Validation   | Must be plain text; line breaks are not allowed.                           |
| Localization       | Must support all display languages.                                        |
| Data Source        | Maintained in CCMS.                                                        |
| Dependencies       | Used as the subtitle or secondary text on the card.                        |

---

#### `carpark_label`

A categorical tag indicating a specific feature, status, or type of parking.

| Specification Item | Details                                                                  |
| ------------------ | ------------------------------------------------------------------------ |
| Field Type         | List (Labels can be configured and saved)                                |
| Example            | EV Charging Available, Free Self-parking                                 |
| Input Validation   | Must select from a predefined list of labels.                            |
| Localization       | Label text must be localized for display.                                |
| Data Source        | Maintained in CCMS, managed under a global taxonomy of carpark features. |
| Dependencies       | Determines the visual tag or badge displayed on each card.               |
