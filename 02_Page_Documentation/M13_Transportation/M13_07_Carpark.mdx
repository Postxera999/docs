## 1.  CarparkInfoCard

```Plain
TRANS_DISPLAY_CarparkInfoCard
```

### **Key Function**

This component displays the primary details of a specific car park, including its name, location, and relevant tags, allowing users to quickly understand its attributes.

### Visual Reference

<FigmaEmbed url="https://www.figma.com/design/EHB41JQIomI0iwPWzaXAkw/-Dev--NGMP-UXUI-Designs?node-id=7729-15997&t=EIrrWlE3DFcM0Te3-4" />

### Field List

| Fields              | Description                                                                                               |
| ------------------- | --------------------------------------------------------------------------------------------------------- |
| carpark_full_name   | Displays the full name of the car park.                                                                   |
| carpark_description | A detailed description of the car park.                                                                   |
| carpark_label       | A label or badge indicating a key feature or status of the car park.                                      |
| latitude            | The geographical latitude coordinate of the carpark's entrance or central point (used for map services).  |
| longitude           | The geographical longitude coordinate of the carpark's entrance or central point (used for map services). |
| location_address    | The full, verifiable street address of the parking facility.                                              |

### Display Logic

- The `CarparkInfoCard` is displayed first on the page, showing key identification and location details.
- `carpark_full_name` appears as the main title.
- If multiple `parking_label` values exist, they will wrap to the next line when exceeding width limits.

### Interaction Logic

- Tapping the **Contact** button triggers the **Contact Modal**. See:  [Contact Info Modal](/01_Global_Components/Contact Info Modal)
- Tapping the **View Map** button sends `latitude`, `longitude`, `location_name`, and `location_address` to the WeChat API to open the native map view.

## 2.  Parking Fee Card

```Plain
TRANS_DISPLAY_ParkingFeeCard
```

### **Key Function**

This component displays the parking fee structure, exclusive benefits, and applicable conditions for using the car park.

### Visual Reference

<FigmaEmbed url="https://www.figma.com/design/EHB41JQIomI0iwPWzaXAkw/-Dev--NGMP-UXUI-Designs?node-id=7729-16229&t=EIrrWlE3DFcM0Te3-4" />

### Display Logic

- Rendered as a structured table or a set of cards.
- The user's membership `tier` determines the displayed fee structure.
- Each item in the `parking_fee_table` represents one rate tier.
- `period`, `penalty`, and `payment_methods` appear below each `tier`.

### Interaction Logic

- This component does not contain interactive elements.

### **CMS Field Specification**

#### `parking_fee_table`

An array of objects used to define specific parking fee tiers.

| **Specification Item** | **Details**                                                                                                                |
| ---------------------- | -------------------------------------------------------------------------------------------------------------------------- |
| Field Type             | JSON Array / Nested Repeater Field                                                                                         |
| Data Structure         | Array of objects, containing nested fields: `tier`, `period`, `penalty`, `payment_methods`.                                |
| Example                | `[{"tier": "Member Tier 1", ...}, {"tier": "Spend $500", ...}]`                                                            |
| Input Validation       | All required nested fields must be populated for each tier object.                                                         |
| Localization           | Content within nested fields (e.g., `tier`, `period.label`, `penalty.label`, `payment_methods` items) must be localizable. |
| Data Source            | Maintained in CCMS                                                                                                         |
| Dependencies           | Dependent on membership data from CRM for `tier` consistency.                                                              |

```JSON
[
        {
                "tier": "xx",
                "period": {
                        "label": "5 hours",
                        "value": "Free"
                        },
                "penalty": {
                        "label": "After 5 hours",
                        "value": "MOP $30 hour"
                },
                "payment_methods": [
                        "Scan The payment...",
                        "Pay with Macau Pass...."
                ]
        }
]
```

---

#### `tier`

Specifies the eligibility criteria or membership level associated with this fee structure.

| **Specification Item** | **Details**                         |
| ---------------------- | ----------------------------------- |
| Field Type             | dropdown/single-selection           |
| Max Length             | N/A                                 |
| Example                | Gold                                |
| Input Validation       | N/A                                 |
| Localization           | Must support all display languages. |
| Data Source            | CRM                                 |

---

#### `period`

Defines the duration of free parking and the corresponding value/status. Contains nested fields: `label` and `value`.

| **Specification Item** | **Details**                                                                 |
| ---------------------- | --------------------------------------------------------------------------- |
| Field Type             | Object containing two nested text fields.                                   |
| Nested Field: `label`  | Description of the period (e.g., "5 hours", "First 3 hours"). Max 30 chars. |
| Nested Field: `value`  | The free status/value (e.g., "Free", "Discounted"). Max 30 chars.           |
| Example                | `{"label": "5 hours", "value": "Free"}`                                     |
| Input Validation       | Both `label` and `value` are required text fields.                          |
| Localization           | Both `label` and `value` must support all display languages.                |
| Data Source            | Nested within the `parking_fee_table` array object.                         |

---

#### `penalty`

Defines the charge applied after the free period expires. Contains nested fields: `label` and `value`.

| **Specification Item** | **Details**                                                                                       |
| ---------------------- | ------------------------------------------------------------------------------------------------- |
| Field Type             | Object containing two nested text fields.                                                         |
| Nested Field: `label`  | Description of when the charge applies (e.g., "After 5 hours", "Subsequent Hours"). Max 40 chars. |
| Nested Field: `value`  | The monetary charge (e.g., "MOP $30/hour", "MOP $10 per 30 mins"). Max 50 chars.                  |
| Example                | `{"label": "After 5 hours", "value": "MOP $30 hour"}`                                             |
| Input Validation       | Both `label` and `value` are required text fields.                                                |
| Localization           | Both `label` and `value` must support all display languages (including currency symbols).         |
| Data Source            | Nested within the `parking_fee_table` array object.                                               |

---

#### `payment_methods`

A list of accepted payment methods available for the carpark.

| **Specification Item** | **Details**                                                    |
| ---------------------- | -------------------------------------------------------------- |
| Field Type             | Array of Text                                                  |
| Max Items              | 5 (Recommended limit for clear display)                        |
| Max Length per Item    | 100 characters                                                 |
| Example                | `["Scan The payment...", "Pay with Macau Pass...."]`           |
| Input Validation       | Items must be descriptive and clearly indicate the method.     |
| Localization           | Each payment method string must support all display languages. |
| Data Source            | Nested within the `parking_fee_table` array object.            |

## 5.  EV Charging Station Brand

```Plain
TRANS_DISPLAY_EVChargingStationBrand
```

### **Key Function**

Displays the logo and name of EV charging providers available in the car park, helping users quickly identify supported charging services.

### Visual Reference

<FigmaEmbed url="https://www.figma.com/design/EHB41JQIomI0iwPWzaXAkw/-Dev--NGMP-UXUI-Designs?node-id=7729-16563&t=EIrrWlE3DFcM0Te3-4" />

### Field List

| Fields          | Description                                  |
| --------------- | -------------------------------------------- |
| evcharging_logo | Logo of the EV charging provider or station. |
| evcharging_name | Name of the EV charging provider or station. |

### Display Logic

- Each charging service appears as a card with:
    - A circular `evcharging_logo` displayed on top.
    - The `evcharging_name` below it (text wraps if too long).
- Not all car parks include EV charging sections.

### Interaction Logic

- No interaction behavior for this component.

### **CMS Field Specification**

#### `evcharging_logo`

Displays the logo of the EV charging service provider or charging station. This logo is critical for brand recognition.

| **Specification Item** | **Details**                                                            |
| ---------------------- | ---------------------------------------------------------------------- |
| Field Type             | Image Upload                                                           |
| File Format            | PNG or JPG                                                             |
| File Size              | Max 500KB                                                              |
| Resolution             | Recommended: 120x120 pixels (Aspect Ratio: 1:1)                        |
| Input Validation       | Must be a valid image file.                                            |
| Localization           | Non-localizable (Typically, logos remain consistent across languages). |
| Data Source            | Maintained in CCMS / EV Charging Module                                |
| Dependencies           | Used in EV Charging station list view and detail page header.          |

#### `evcharging_name`

Displays the name of the EV charging service provider or charging station. This is the official and user-facing name of the service location or brand.

| **Specification Item** | **Details**                                                                                                                    |
| ---------------------- | ------------------------------------------------------------------------------------------------------------------------------ |
| Field Type             | Text input (single line)                                                                                                       |
| Max Length             | 30 characters                                                                                                                  |
| Characters Limit       | None                                                                                                                           |
| Example                | Tesla Supercharger                                                                                                             |
| Input Validation       | Must not contain HTML, script tags, or unrecognized special symbols. No emojis. Leading/trailing spaces automatically trimmed. |
| Localization           | Must support all display languages.                                                                                            |
| Data Source            | Maintained in CCMS / EV Charging Module                                                                                        |
| Dependencies           | Used in search results, EV Charging station list view, and detail page header.                                                 |
