The **Venue Directory** page lists all entertainment venues and shows available within the Galaxy platform. It serves as a discovery hub for users to browse events, access countdowns, and view ticket availability.

This page references data from the **Event Directory content type** within the Entertainment module, configured in the **CCMS** to manage venue listings, sorting, and event display order.

3. ## ShowCard
    

```Plain
ENT_DISPLAY_ShowCard
```

### Key Function

Displays a list of each upcoming Entertainment Event within the Venue Directory. Each card links directly to the **Entertainment Event Detail** page.

### Visual Reference

<FigmaEmbed url="https://www.figma.com/design/EHB41JQIomI0iwPWzaXAkw/-Dev--NGMP-UXUI-Designs?node-id=7729-27781&t=EIrrWlE3DFcM0Te3-4" />

  

### Field List

|   |   |
|---|---|
|Fields|Description|
|subcategory|Entertainment Category, e.g. Fan Meeting, Concert/Music, Cultural, Shows, Others. One show can have more than one category but only displays the primary category as the tag|
|hero_image|Hero/thumbnail image of each entertainment event|
|title_text|Entertainment Event title|
|show_time_specification|All date and times for the show|
|countdown_label_text|Countdown description text|
|countdown_start_in_hours|Countdown time to when the sale starts|
|price_schedule_specification|From price (lowest available price), in MOP|
|offers|No. of linked special offers attached to the Entertainment Event|
|label_specification|Displays tags to indicate priority sale availablity, membership exclusivity, or other benefits|
|sale_status|Ticketing sale status of the entertainment event: sale not started, presale, on sale, or sold out|

### Display Logic

- The primary `subcategory` is displayed as a tag on the left corner of each `hero_image`. Each Entertainment Event will only display one `subcategory` on this card.
    
- The Entertainment Event `title_text` is displayed in a maximum of 2 lines. If the title is longer, it is truncated.
    
- The `show_time_specification` is displayed in full under the `title text`.
    
- The `price_schedule_specification` is fixed to the bottom right of the card and displays the lowest ticketing price with an approximate conversion.
    
    - Main pricing displayed as: `[From {lowest price}]`
        
    - Conversion to CNY displayed below as: `[Approx. {lowest price CNY conversion}]`
        
    - If `pricing_availability_override` is enabled, replace `[From {Price}]` and `[Approx. {CNY Price}]` with [Pricing Not Yet Available]

### 3.1 Countdown Timer

```Plain
GLOBAL_STATUS_CountdownTimer
```

#### Key Function

- Displays a countdown timer indicating the time remaining before general ticket sales open.
    
- Linked to the event’s Sale Schedule Item.
    

#### Visual Reference

<FigmaEmbed url="https://www.figma.com/design/EHB41JQIomI0iwPWzaXAkw/-Dev--NGMP-UXUI-Designs?node-id=7771-14199&t=EIrrWlE3DFcM0Te3-4" />

  

#### Display Logic

If a **countdown timer** is configured for the Entertainment Event, display the countdown timer on top of the `hero_image`, with `countdown_label_text`on the left, and live countdown on the right.

- Countdown format in real-time: DD:HH:MM:SS
    
- Once the time of the sale starts, the component is hidden

### 3.2 CTA Button

```Plain
GLOBAL_ACTION_CTA Button
```

#### Key Function

- Allows user to perform actions in relation to the event's sale status e.g. Set Reminder
    

#### Visual Reference

<FigmaEmbed url="https://www.figma.com/design/EHB41JQIomI0iwPWzaXAkw/-Dev--NGMP-UXUI-Designs?node-id=7771-14198&t=EIrrWlE3DFcM0Te3-4" />

  

#### Display Logic

- Reminder CTA button:
    
    - If the sale for an Entertainment Event has not started and reminder notifications are available, display Set Reminder as an active CTA button.
        
    - If the user has not subscribed to any sale reminder notification messages and a notification message is available, the button is displayed in its active state.
        
- Sold out button:
    
    - If the `sale_status` is "Sold out", the button is disabled and inactive as a display only CTA.
        

  

#### Interaction Logic

- If the button is active, on tap, opens the default WeChat notification subscription pop up for users to select which sales they want to subscribe to.
    
- When the reminder is successfully subscribed to, display a toast message to indicate that the reminder has been added
    
### 3.3 Access Label

```Plain
ENT_DISPLAY_AccessLabel
```

#### Key Function

- Shows when the user has exclusive access to the priority sale, or entertainment event due to their member tier to let them know they have this privilege.
    

#### Visual Reference

<FigmaEmbed url="https://www.figma.com/design/EHB41JQIomI0iwPWzaXAkw/-Dev--NGMP-UXUI-Designs?node-id=7771-14196&t=EIrrWlE3DFcM0Te3-4" />

  

#### Display Logic

- There are several levels of access for each Entertainment Event:
    
    - Member-only access:
        
        - The Entertainment Event is only bookable for certain membership tiers, other tiers will see the event details only, but no information on how to purchase a ticket.
            
    - Priority Sale access:
        
        - By membership tier:
            
            - Locked state: If the user's member tier does not match the Priority Sale criteria, they will see the eligible tiers for priority access
                
            - Unlocked state: If the user's member tier matches and they have the corresponding coupon ID for priority access, they will see that their access is unlocked
                
        - By other criteria
            
            - Locked state: If the user does not match the Priority Sale criteria, they will see that access is locked
                
            - Unlocked state: If the user's matches the Priority Sale criteria and they have the corresponding coupon ID for priority access, they will see that their access is unlocked
