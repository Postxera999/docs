### Page Overview

The **Entertainment Event** **Detail** page provides complete information about each concert or entertainment event within the Galaxy platform.

It displays ticket details, seating information, membership benefits, FAQs, purchase instructions etc. This page references data from the **Entertainment Event content type** in the CCMS.

- Displayed at the top of the Entertainment Event Page

    - Please refer to that section for color picker details. [View](/01_Global_Components/Color_Picker_Function)

    - The colour contrast is then calculated and determined if it is a dark or light colour

- The height of the card is dynamic and responsive, determined by the number of lines in the `title_text`. The full `title_text` should always be displayed.
    
- The `price_schedule_specification` is fixed to the bottom right of the card and displays the lowest ticketing price with an approximate conversion.
    
    - Main pricing displayed as: `[From {lowest price}]`
        
    - Conversion to CNY displayed below as: [Approx. `{lowest price CNY conversion}`]`
        
    - If `pricing_availability_override` is enabled, replace [From {Price}] and [Approx. `{CNY Price}`] with [Pricing Not Yet Available]

### 3.4 Sale Schedule

```Plain
ENT_DISPLAY_SaleSchedule
```

**Key function:**

Galaxy occasionally implements phased ticket sale strategies to manage demand and provide targeted access for high-interest Entertainment Events. These strategies often include multiple exclusive pre-sale periods followed by a public sale.

  

Sale phases may include:

- _Galaxy-Managed Pre-Sales_: Pre-sale access is controlled through Galaxy’s ticketing platform. Entry is granted via access codes embedded in NG CRM coupons, allowing targeted groups (e.g. Diamond-and-above-tier members) to purchase tickets before public availability.
    
- _Externally Managed Pre-Sales_: Some pre-sales are handled outside Galaxy’s ticketing ecosystem, such as through Damai or Ctrip. These pre-sales utilize alternative external mechanisms (e.g. an artist’s Weverse fan club site) for distributing access codes and, as such, cannot be managed through NG CRM and WCMP.
    
- _Exclusive Member Sales_: Similar to pre-sales, these events are restricted to specific member groups and require access code coupons to complete the purchase.
    
- _Public Sales_: Tickets are made available to all users without access restrictions. Sales may occur across one or multiple ticketing platforms, depending on the event’s distribution strategy.
    

#### Field List

| Fields          | Description                                                                                                                                            |
| --------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------ |
| title_text      | Each Sale Schedule is assigned a meaningful name to help content managers easily identify and associate it with the corresponding Entertainment Event. |
| sale_start_date | Defines the start date and time for ticket sales within a Sale Schedule Item.                                                                          |
| sale_end_date   | Defines the end date and time for ticket sales within a Sale Schedule Item.                                                                            |
| subtitle_text   | The subtitle text is displayed below the Sale Schedule window.                                                                                         |

#### Display Logic

- Each Sale Schedule Window is displayed as its own line item.
    
- `title_text` is displayed as the left-column header
    
- `sale_start_date` / `sale_end_date`are displayed as the primary text style in the right-column.
    
- `subtitle_text` is displayed as the secondary text style if configured
    

  

- _Sale Schedule Item Name_ – Each Sale Schedule Item must be assigned a meaningful name to help content managers easily identify and associate it with the corresponding Entertainment Event.
    
- _Sale Start Date_ – Defines the start date and time for ticket sales within a Sale Schedule Item. This value is not only displayed across various pages but also used to sequence Sale Schedule Items, determining the order in which associated labels and promotional banners are presented.
    
- _Sale End Date_ – Defines the end date and time for ticket sales within a Sale Schedule Item. Sale schedule dates and times are displayed according to the following rules:
    
    - _No end date and time_ – Starting Month Day, Year Time (e.g. Starting November 15, 2025 14:00)
        
    - _Same start and end date with different times_ – Month Day, Year Time1 - Time2, Time3 (e.g. November 15, 2025 14:00 – 16:00)
        
    - _Different dates_ –MM.DD.Year HH:MM – MM.DD.Year HH:MM (e.g. 12.30.2025 12:00 – 01.02.2026 19:00)
        
    - _Multiple dates and times_ – Display on separate lines.
        
- _Title Text_ – The title text is displayed in multiple places on the Entertainment Event page:
    
    - _Sale Schedule Window_ – Appears at the top of the page to indicate the current or upcoming sale phase.
        
    - _How to Book Window_ – Shown when the Sale Schedule Item Type is set to Pre-Sale or Member Exclusive Sale.
        
- _Subtitle Text_ – The subtitle text is displayed below the title in the Sale Schedule window.

### Display Logic

- If there is more than one booking type configured, the `ticketing_type_title` is displayed as tabs.
    
- If only one type is configured, hide the tabs.
    
- If the event is locked to certain membership tiers, hide the 'How to book' section.

- The `ticketing_type_description` for the booking type is placed at the top of the section.
    
- For online ticket purchases, a `default_ticketing_url` with a corresponding `ticketing_url_title` can be configured.
    
- More than one `default_ticketing_url`can be configured for one Entertainment Event, the subsequent link components will be placed on below.
    
- The `default_ticketing_url` links will only be displayed in one line, if it exceeds one line, the url text will be truncated.
    

  

Book at Counter

- The `ticketing_type_description` for the booking type is placed at the top of the section.
    
- The content references the `box_office_reference` specification, where the `box_office_name` and `location_specification` is configured. Each counter location is displayed as a line item in a table format.
    

  

### Interaction Logic

- When tapping the [Copy link] CTA button next to the `default_ticketing_url`, the configured URL will be copied to clipboard and the CTA button copy changes to [Copied]
    
- To switch between booking types, the tabs at the top of the section will switch the content configured under each location.
