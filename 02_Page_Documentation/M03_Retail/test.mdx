## **1. Transport Service Selector**

```
TransportServiceSelector
```

### **Key Function**

This section serves as a navigation component at the top of the page, allowing users to select between different types of transportation services (e.g., Shuttle Bus, Express Limousine, etc.). It presents these options in tab or button format, enabling users to quickly switch and view details of different transport services.

### Visual Reference

**Figma reference to be added**

### Field List

| Fields                   | Description                                                                  |
| ------------------------ | ---------------------------------------------------------------------------- |
| `transportation_options` | Key-value list that configures various transportation service options.       |
| `transportation_title`   | Display name shown on the selector, such as “Shuttle Bus” or “Limo Service.” |
| `transportation_type`    | Type identifier mapped to the specific transportation card style.            |
| `transportation_status`  | Enable/disable status of each transport service.                             |

### Display Logic

* The component is located at the top of the page, right below the page title, and displayed as a horizontally scrollable card list.
* It renders all **active** button cards based on the configuration of `transportation_options` and `transportation_status`.
* The display order of the buttons follows the `transportation_order` field.
* The currently selected `transportation_title` is highlighted or marked as active.
* When the number of `transportation_options` exceeds the display width, users can scroll horizontally.
* Component visibility is controlled by the `transportation_status` value.

### Interaction Logic

* When users swipe left on the navigation, the page content remains unchanged.
* When users select a navigation item, the main content area dynamically loads and displays the corresponding `transportation_type`.
* When the second or subsequent navigation items are selected, the container automatically scrolls left until the last navigation item aligns with a 16px right margin.

## **Transport Schedule Form**

```
TransportScheduleForm
```

### **Key Function**

This component provides an input form where users enter key details such as **departure**, **destination**, and **date** to search for available transport options.

### Visual Reference

**Figma reference to be added**

### Field List

| Fields       | Description                                             |
| ------------ | ------------------------------------------------------- |
| station_name | Name of the station.                                    |
| station_type | Type of station, distinguishing shuttle bus and others. |

### Display Logic

* Placed below the **TransportServiceSelector**.
* Includes fields for **departure** and **destination** (for shuttle bus type stations).
* Display logic examples:

  * If user is in Galaxy → show “From Galaxy to StarWorld”.
  * If user is in StarWorld → show “From StarWorld to Galaxy”.
  * If user’s location cannot be obtained or is outside Galaxy/StarWorld → show “From Border Gate to Galaxy/StarWorld”.
* Selecting a field opens a bottom-up modal (**Location Selector Modal**).

### Interaction Logic

* Clicking **Departure/Destination** triggers the **LocationSelectorModal**.
* Upon selection, the modal fills `from_station_id` or `to_station_id`.
* Clicking **View Schedule**:

  * Submits a route query based on form selections.
  * On success, scrolls to results or navigates to `[M13_02 Shuttle Route Details](link)`.

## **Station Selector Modal**

```
StationSelectorModal
```

### **Key Function**

A **popup modal** that appears when users tap **Departure** or **Destination** in the **TransportScheduleForm**, providing a structured list of stations or locations.

### Visual Reference

**Figma reference to be added**

### Field List

| Fields       | Description                                    |
| ------------ | ---------------------------------------------- |
| station_name | Station name.                                  |
| station_type | Type of station (e.g., shuttle bus or others). |

### Display Logic

* Appears as a **bottom modal** overlaying the page.
* Contains station list selectors.
* Includes two types: **Departure Selector** and **Destination Selector**.
* Each list item shows `station_name`.
* Can be filtered or grouped by `station_type` or `is_in_galaxy`.

### Interaction Logic

* Users can **search**, **scroll**, or **select** a location.
* Upon selection, the modal closes and fills the corresponding field in **TransportScheduleForm** (`station_id` or `howtofind_id`).
