## 2 Property Card List

  

```Plain
GLOBAL_PNAV_NavigationBar
```

  

### **Key Function**

  

This component appears after the user taps the **[Property Selector](/01_Global_Components/Property_Selector)**.

It displays a list of property cards, allowing users to view and select different properties (locations) through large, visually rich cards. Each card provides a quick summary of the property’s offerings via category tags.

  

### Visual Reference

  

<FigmaEmbed url="https://www.figma.com/design/dogOPeGxUe0sjpG2jGnayE/NG-MP-UI-Phase-1?node-id=13480-12076&t=Xla5s1etoe8gKRgp-4" />

  

### Field List

  

|   |   |
|---|---|
|Fields|Description|
|`property_list`|A list container holding all property card data.|
|`property_id`|Unique identifier of the property used to load the homepage.|
|`property_name`|Display name of the property (e.g., Galaxy & Broadway).|
|`active_property`|Indicates the currently selected property.|
|`background_image`|Background image used in the card.|
|`category_list`|List of service/facility categories available under the property.|

  

### Display Logic

  

- `PropertyCardList` displays all available properties in a vertical card list, with **Galaxy** listed first and **StarWorld** below it.
    
- Each Property is displayed as a **full card** containing the background image, name, state tag, and category tags.
    
- The currently selected property card displays a “Current” tag, driven by the `active_property` field.
    
- The **Current** property card appears brighter, while non-current ones are darkened with a semi-transparent black overlay.
    
- `category_list` is **automatically generated** based on the modules configured for the property.
    

Categories appear as horizontally placed tags/buttons, wrapping to a new line when overflowing.

  

### Interaction Logic

  

- When the user taps **any non-current** property card:
    
      
    
    - **Action Triggered:** Switch property using the tapped card’s `propertyId`.
        
    - **Expected Result:** The system loads the new property homepage and updates its state tag to “Current”.

**Description:** Indicates the current property selected by the user.

  

|   |   |

2. User’s **most recent booking** is at StarWorld
    
      
    

Otherwise, the default property is `Galaxy`.

## 4 Logged Out Coupon CTA Banner

  

```Plain
HOME_DISPLAY_LoggedOutCouponCTABanner
```

  

### Key Function

  

This component displays **when the user is logged out** and has exclusive coupons available to claim, encouraging signup/login and improving conversion.

  

### Visual Reference

  

<FigmaEmbed url="https://www.figma.com/design/dogOPeGxUe0sjpG2jGnayE/NG-MP-UI-Phase-1?node-id=13480-12077&t=Xla5s1etoe8gKRgp-4" />

  

### Field List

  

|   |   |
|---|---|
|Fields|Description|
|`coupon_count`|Number of coupons available to claim.|
|`coupon_total_value`|Total value of coupons (e.g., MOP 500).|

  

### Display Logic

  

- **Placement:** Below the homepage hero image, under the navigation bar.
    
- **Condition:** Displayed only when:
    
      
    
    - User is logged out, AND
        
    - There are claimable coupons.
        
- **Priority:** If `couponCount > 0`, this component **overrides** any generic signup/login component.
    
      
    

### Interaction Logic

  

- **Tapping “Sign Up” button:**
    
      
    
    - Opens the login/registration flow.
        
    - After successful signup/login:
        
          
        
        - User returns to homepage.
            
        - This banner disappears (as the user is now logged in).

|---|---|
|Specification Item|Details|
|Field Type|Integer|
|Max Value|99 (show as 99+ if exceeding)|
|Input Validation|Must be an integer > 0|
|Localization|Number is universal; text such as “coupons to collect” must be localized|
|Data Source|Dynamically calculated for anonymous user|

  

#### `coupon_total_value`

  

|   |   |
|---|---|
|Specification Item|Details|
|Field Type|Text (numeric + currency unit)|
|Max Length|20 characters|
|Example|`MOP 500`|
|Input Validation|Must include **currency unit** + **value**|
|Localization|Format may vary by locale (e.g., `$500 MOP`)|
|Data Source|Derived from coupon value fields|

## 10 Complete Profile Incentive Card

  

```Plain
HOME_DISPLAY_CompleteProfileIncentiveCard
```

  

### **Key Function**

  

This component is a **high-priority notification/incentive card** designed to remind and **motivate users** to complete certain tasks (e.g., completing their personal profile) in order to earn rewards.

Its objective is to improve the completeness of user profiles while enhancing user experience and engagement through incentives.

  

### Visual Reference

  

<FigmaEmbed url="https://www.figma.com/design/dogOPeGxUe0sjpG2jGnayE/NG-MP-UI-Phase-1?node-id=13240-12726&t=Xla5s1etoe8gKRgp-4" />

  

### **Field List**

  

|   |   |
|---|---|
|Fields|Description|
|`expireTimeText`|Text description of reward/task expiration (e.g., "Expire in 1 month").|
|`incentiveMessage`|Incentive message encouraging users to complete their profile.|
|`currentProgress`|Number of steps the user has already completed.|
|`totalProgress`|Total number of steps required.|

  

### **Display Logic**

  

- **Position:** Displayed below the `UserMemberCard` component as a full-width standalone card. Lower priority than `My Stay Widget`.
    
- **Close Button:** An “X” icon must be shown at the top-right corner, allowing users to temporarily hide the card.
    
- **Top Hint:** The top-left area displays the **expiration hint** (`expireTimeText`).
    
- **Main Body:** The central section displays the **incentive message** (`incentiveMessage`) in bold.
    
- **Progress Bar:** The bottom section shows the profile completion progress.
    
      
    
    - Visual element: A `Progress Bar`
        
- **Action Button:** Bottom-right displays the CTA button prompting the user to start the Edit Profile action.
    
      
    

### **Interaction Logic**

  

- **Close Button (X):** Temporarily hides the card.
    

==This hidden state resets when the user reopens the app, unless the task has been completed or the reward has expired.==

- **Click CTA Button (Edit Profile):** Navigates to the `[WIP]` profile-editing page.
    
- **Task Completion:** If `currentProgress` equals `totalProgress`, the card will no longer be displayed.

## **11 Upgrade Incentive Card**

  

```Plain
HOME_DISPLAY_UpgradeIncentiveCard
```

  

### **Key Function**

  

This component is a **call-to-action card** that prompts users who have already met the upgrade requirement but **have not yet completed** the final upgrade steps (e.g., filling in profile details).

It simplifies the upgrade entry point and motivates users to complete the final step to enjoy the benefits of the next tier.

  

### Visual Reference

  

<FigmaEmbed url="https://www.figma.com/design/dogOPeGxUe0sjpG2jGnayE/NG-MP-UI-Phase-1?node-id=13246-13192&t=Xla5s1etoe8gKRgp-4" />

  

### **Field List**

  

|   |   |
|---|---|
|Fields|Description|
|`upgrade_title`|Main title text of the upgrade card|
|`upgrade_subtitle`|Subtitle / instructional text|

  

### **Display Logic**

  

- **Prerequisite:** Displayed when the user is eligible for an upgrade but hasn’t completed the required profile info.
    
- **Position:** Displayed below the `UserMemberCard` and lower in priority than `My Stay Widget` and `Complete Profile Incentive Card`.
    
- **Dynamic Style:** The **background/style** must reflect the next level (e.g., Current = Gold → Display Platinum style when user is eligible but incomplete).
    
      
    

### **Interaction Logic**

  

- **CTA Button (Finish Upgrade):** Navigates to the `[WIP]` profile-completion page.
    
- **After Completion:** The card will no longer be displayed once the user completes the profile information.

|Example|Eligible for Upgrade|
|Localization|Must support all display languages.|
|Data Source|CMS or fixed system text.|
