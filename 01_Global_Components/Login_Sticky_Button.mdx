The **Login Sticky Button** is a floating global call-to-action prompting users to **log in** to access member-exclusive features. It remains visible on public pages until the user is authenticated.

---

## Display Logic

- Floating global CTA for unauthenticated users.
    
- Hidden automatically once logged in.
    
- Prompts users to sign in for member-exclusive actions such as **Set Reminder**, **Priority Sale**, or **Exclusive Content**.

## Key Functional Features

| No. | Feature                | Description                                                      |
| --- | ---------------------- | ---------------------------------------------------------------- |
| 1   | Floating CTA           | Stays visible as user scrolls, improving accessibility.          |
| 2   | Conditional Visibility | Displays only when user is not logged in.                        |
| 3   | Dynamic Label          | Text changes depending on context (e.g., “Log In to View More”). |
| 4   | Global Application     | Shared across all entertainment modules.                         |

---

## CMS Field Specification

| Field Name           | Description                              |
| -------------------- | ---------------------------------------- |
| Button Label         | Default: “Log In to View More”.          |
| Redirect URL         | Destination for authentication redirect. |
| Visibility Condition | Boolean (logged in/out state).           |

---

## Field Mapping Details

### `button_label`

| Specification Item | Details                          |
| ------------------ | -------------------------------- |
| Field Type         | Text                             |
| Input Validation   | Optional                         |
| Localization       | Supports all display languages   |
| Data Source        | CCMS                             |
| Example            | “Log in to access Priority Sale” |

### **`redirect_url`**

| Specification Item | Details                             |
| ------------------ | ----------------------------------- |
| Field Type         | URL                                 |
| Input Validation   | Required                            |
| Data Source        | Global Auth Config                  |
| Example            | /login?redirect=/events/galaxy-live |

### **`visibility_condition`**

| Specification Item | Details               |
| ------------------ | --------------------- |
| Field Type         | Boolean               |
| Input Validation   | System controlled     |
| Data Source        | User session state    |
| Dependencies       | Hidden when logged in |
| Example            | false (logged in)     |
